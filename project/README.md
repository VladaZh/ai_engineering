# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект должен демонстрировать применение методов и инструментов инженерии ИИ: работу с данными, модели, пайплайны, сервис, эксперименты и (по возможности) воспроизводимость.


---

## 1. Паспорт проекта

- **Название проекта:** `Fitness Classifier`
- **Автор:** `Жуйкова Владислава Анатольевна`
- **Группа:** `ИКБО-10-24`
- **Контакт:** `@prtkvvktrgrvn`

- **Краткое описание (2-4 предложения):**  

> Проект посвящён построению сервиса бинарной классификации физической формы.  
> Используется датасет "Fitness Classification Dataset (Synthetic)" с Kaggle (2000 записей, 7 признаков), методы предобработки данных, логистическая регрессия в качестве baseline и случайный лес как улучшенная модель, а также визуализация метрик и важности признаков. 
> Результат – REST API, который по параметрам пользователя возвращает прогноз и вероятность принадлежности к классу «в форме» (is_fit).

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - модули для работы с данными;
  - обучение моделей;
  - запуск сервиса/скриптов.
- `data/` – демонстрационные/учебные данные:
  - **только открытые, синтетические или обезличенные данные;**
  - при необходимости – небольшие выборки для быстрого запуска.
- `configs/` – конфигурационные файлы:
  - `.yaml`
  - `.env.example`
- `tests/`:
  - test_classifier - проверяется корректность работы класса FitnessClassifier.
  - test_preprocessor - проверяется корректность и надёжность класса DataPreprocessor.
  - test_service - проверяется работоспособность FastAPI-сервиса.
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов. Файлы в данной папке генерирутся в процессе работы программы.

Если структура проекта отличается от базовой, опишите это здесь.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10` (уточните при необходимости).

### 3.2. Установка окружения

Примерный сценарий (можно адаптировать под ваш проект):

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение (опционально, но рекомендуется)
python -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt
```

## 4. Как запустить проект

### 4.1. Запуск обучения модели

```bash
cd project
source .venv/bin/activate
jupyter execute notebooks/01_eda_baseline.ipynb
```

или, если датасет загружен:

```bash
cd project
source .venv/bin/activate
python -m src.train
```

### 4.2. Запуск сервиса (API/веб-интерфейс)

```bash
cd project
source .venv/bin/activate
uvicorn src.service.app:app --reload --host 0.0.0.0 --port 8000      
```

или

```bash
cd project
source .venv/bin/activate
python -m src.service
```

Опишите:

- сервис поднимается на 8000 порту;
- ключевые эндпоинты: /health - проверка работоспособности сервиса, /predict - основной функционал (классификация физической формы) и /features - помощь клиентам в заполнении данных (вывод всех учитываемых параметров и единиц измерения);
- протестировать работоспособность можно с помощью команды:
```bash
curl http://localhost:8000/health
```

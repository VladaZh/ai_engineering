# HW08-09 – PyTorch MLP: регуляризация и оптимизация обучения

## 1. Кратко: что сделано

- **Датасет:** EMNIST (split="balanced") — Вариант B. Выбран для демонстрации работы методов регуляризации на задаче с 47 классами.
- **Часть A (регуляризация):** Сравнивались базовая модель, Dropout (p=0.3), BatchNorm и EarlyStopping для борьбы с переобучением.
- **Часть B (оптимизация):** Исследовалось влияние экстремальных значений learning rate и сравнение Adam vs SGD+momentum+weight decay.

## 2. Среда и воспроизводимость

- **Python:** 3.12.12
- **torch / torchvision:** 2.10.0 / 0.25.0
- **Устройство (CPU/GPU):** CPU
- **Seed:** 42
- **Как запустить:** открыть `HW08-09.ipynb` и выполнить Run All.

## 3. Данные

- **Датасет:** EMNIST Balanced
- **Разделение:** train=90 240 (80%) / val=22 560 (20%) / test=18 800
- **Трансформации:** ToTensor(), Normalize(mean=0.5, std=0.5)
- **Комментарий:** 47 классов (строчные и заглавные буквы), изображения 28×28 в оттенках серого. Задача сложнее MNIST из-за большего числа классов и визуальной схожести некоторых символов.

## 4. Базовая модель и обучение

- **Модель MLP:** 2 скрытых слоя [512, 256], активация ReLU, выходной слой 47 классов
- **Loss:** CrossEntropyLoss
- **Базовый Optimizer:** Adam (lr=1e-3)
- **Batch size:** 128
- **Epochs (макс):** 20 (для E1-E3), 30 (для E4 с EarlyStopping)
- **EarlyStopping:** patience=5, metric=val_accuracy

## 5. Часть A (S08): регуляризация (E1-E4)

- **E1 (base):** 2 скрытых слоя, без Dropout/BatchNorm
- **E2 (Dropout):** как E1 + Dropout(p=0.3) после каждого скрытого слоя
- **E3 (BatchNorm):** как E1 + BatchNorm1d между Linear и ReLU
- **E4 (EarlyStopping):** лучший из (E2/E3) по val_accuracy + EarlyStopping (patience=5)

## 6. Часть B (S09): LR, оптимизаторы, weight decay (O1-O3)

- **O1:** LR слишком большой (Adam, lr=1e-1)
- **O2:** LR слишком маленький (Adam, lr=1e-5)
- **O3:** SGD+momentum (momentum=0.9) + weight_decay=1e-4 (lr=1e-2)

## 8. Анализ

На графиках E1 (base) видно умеренное переобучение: train loss продолжает снижаться до 0.20, а val accuracy стабилизируется на уровне ~83.5–84.0% после 10-й эпохи. Разрыв между train и val метриками составляет ~2–3%.

Dropout (E2) показал лучший результат по сравнению с базой: best val accuracy 84.79% против 83.99% у E1. Это подтверждает, что случайное отключение нейронов помогает модели обобщать.

BatchNorm (E3) дал наилучший результат в части A: 85.20% val accuracy на 13-й эпохе, при этом сходимость была быстрее — уже на 3-й эпохе достигнут уровень, который E1 показал только к 10-й.

EarlyStopping (E4) остановил обучение на 14-й эпохе, когда val accuracy перестал улучшаться в течение 5 эпох подряд. Это предотвратило возможное переобучение и сохранило модель с лучшими весами. Итоговая точность на test-наборе составила 84.56%.

Эксперименты с learning rate (O1/O2) наглядно демонстрируют важность подбора этого гиперпараметра:
- **O1 (lr=1e-1):** val accuracy колеблется в диапазоне 75–81%, loss не монотонен — модель "прыгает" вокруг минимума.
- **O2 (lr=1e-5):** accuracy растёт линейно и медленно (49% → 73% за 8 эпох) — шаг оптимизации слишком мал для эффективного обучения.

SGD+momentum+weight_decay (O3) показал результат, сопоставимый с Adam: 85.38% val accuracy. Momentum помог ускорить проход через "плато", а weight decay (1e-4) слегка снизил переобучение — разрыв train/val loss меньше, чем в E1.

Для EMNIST с 47 классами архитектура из двух скрытых слоёв с BatchNorm оказалась оптимальным компромиссом между ёмкостью модели и риском переобучения.

## 9. Итоговый вывод

Базовым конфигом для задач классификации изображений с MLP я бы взял: **2–3 скрытых слоя + BatchNorm + Dropout(p=0.2–0.3) + EarlyStopping(patience=5)**. Это сочетание обеспечивает стабильную сходимость и хорошее обобщение без избыточной ручной настройки.

Дальнейшие улучшения:
1. Добавить **Learning Rate Scheduler** (например, ReduceLROnPlateau) для автоматической адаптации LR в процессе обучения.
2. Попробовать **свёрточную архитектуру (CNN)**, которая учитывает пространственную структуру изображений и обычно показывает лучшие результаты на задачах компьютерного зрения.
